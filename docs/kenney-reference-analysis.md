# Kenney Reference Analysis

This note records the source basis for the Kenney-informed Mochi Social game asset refresh. It is a reference and art-direction document only: no Kenney files are copied into the repo.

## Source Hierarchy

- Official Kenney support/license page: Kenney says game assets on the asset pages are public domain licensed, CC0, usable in commercial projects, and attribution is not required.
- Official Kenney asset pages:
  - Tiny Town: RPG, roguelike, town, overworld, map, pixel tags; 16x16 tile size; 130 files; CC0.
  - RPG tag page: lists Tiny Town, Roguelike Characters, Fantasy UI Borders, Tiny Dungeon, Sketch Town, RPG Urban Pack, and related RPG-compatible packs.
  - UI Pack: buttons, panels, sliders, interface tags; 430 files; CC0.
  - Input Prompts: controls/interface tags; 64x64 tile size; 1500 files; CC0.
- Community credibility signals:
  - Steam top-rated Kenney Game Assets reviews describe the packs as strong for prototyping, useful as final art, broad in sprites/objects/tiles, and royalty-free/commercial-friendly.
  - Defold Asset Portal lists Kenney Free Game Assets as 30,000+ assets, CC0 1.0 Universal, and suitable for HTML5 among other platforms.

## Closest Fits For Mochi Social

1. Tiny Town
   - Best fit for the current town tilesheet because it is RPG/town/overworld/pixel oriented.
   - Inspires compact readable ground, path, water, town landmark, and map-tile silhouettes.
2. Roguelike Characters
   - Best fit for compact player/NPC readability.
   - Inspires large-head/small-body silhouettes, chunky outlines, and simple animation clarity.
3. Fantasy UI Borders / UI Pack / UI Pack Adventure
   - Best fit for HUD, panels, action buttons, and game-native interface language.
   - Inspires simple beveled panels and readable contrast, not copied UI files.
4. Input Prompts
   - Best fit for future keyboard/controller prompt glyphs.
   - Inspires clear icon-first prompt treatment and consistent control affordances.
5. Tiny Dungeon / Sketch Town / RPG Urban Pack
   - Secondary references for props, market/trade structures, dungeon-like shrines, and environment accents.

## Translation To Mochi Social

Kenney's strongest lesson for this project is not visual complexity; it is small-scale clarity. The refreshed Mochi Social assets keep the Cozy Wushu theme but borrow these general principles:

- crisp icon-like silhouettes;
- high contrast at gameplay zoom;
- compact readable tiles and props;
- consistent palette across asset families;
- light bevels, rim highlights, and contact shadows;
- no baked text, logos, or third-party marks;
- no real-value or production-chain visual promises.

## Implementation Notes

- Runtime assets remain generated by `apps/game/scripts/generate-assets.mjs`.
- Current runtime files, dimensions, and paths are unchanged.
- No Kenney asset files were downloaded, copied, traced, embedded, or vendored.
- All regenerated PNGs remain original project-authored procedural art and ship under the repository MIT license.
